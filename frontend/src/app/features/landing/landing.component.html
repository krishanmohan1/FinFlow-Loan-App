Applicant flow

Exactly what a borrower does inside FinFlow.

01

Create account

Register with a username and password, then enter your applicant workspace immediately after account creation.

02

Profile and workspace

See your account summary, application progress, loan status cards, and the next recommended actions in one place.

03

Apply in guided steps

Choose the product, enter the amount, and explain the purpose. FinFlow sends the application into the review pipeline.

04

Upload documents

Attach Aadhaar, PAN, salary slips, bank statements, or property proof against the right loan record.

05

Track the lifecycle

Watch statuses such as pending, under review, approved, or rejected without guessing what happened in the background.

06

Receive decision

Admins record the decision, remarks, and sanction details so the applicant sees a complete and transparent outcome.

What we ask for

Only the information the current backend actually uses.

Account details

  • Username
  • Password

Account creation is intentionally lightweight so the borrower can enter the workspace quickly.

Loan details

  • Loan type
  • Requested amount
  • Purpose of borrowing

The application form is aligned to the working backend contract instead of collecting unsupported fields.

Document details

  • Linked loan ID
  • Document type
  • File upload

Applicants upload only what the document service can currently store and verify end to end.

Admin workflow

Where the operations team works.

Admin users sign in with the seeded admin account and land in a dedicated command center for loans, documents, users, and reporting. The frontend now makes that path explicit instead of hiding it behind role-based routing.

Loan decisionsLoad a case, set approved or rejected, attach remarks, and save sanction details.
Document checksVerify or reject borrower uploads with comments that feed back into the borrower journey.
User controlsReview users, update role or active status, and keep oversight in one secure area.
