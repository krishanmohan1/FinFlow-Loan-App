What we ask for

The information and documents FinFlow uses to process a real lending journey.

Borrower profile

  • Full name, email, and mobile number
  • Date of birth and occupation
  • Annual income
  • Residential address, city, state, and PIN code
  • Username and password

FinFlow captures the borrower profile upfront so the workspace, admin review, and reporting stay aligned.

Loan request

  • Loan type
  • Requested amount
  • Preferred tenure in months
  • Purpose of borrowing

The application form reflects the real backend contract and supports a complete digital intake instead of a placeholder demo form.

Supporting documents

  • Linked loan ID
  • Document type such as Aadhaar, PAN, salary slip, bank statement, or property proof
  • Secure file upload

Applicants attach files against the selected loan, and the verification team can review them with status and remarks.

Admin workflow

Where the FinFlow operations team works.

Admin users sign in to a dedicated command center for loans, documents, users, staff onboarding, and reports. The same platform that serves applicants also gives credit teams a secure place to manage real lending work.

Document checksVerify or reject borrower uploads with comments that feed back into the borrower journey.
Staff controlsReview users, onboard internal admin accounts, and manage role or active status in one secure workspace.
